# 3분 데모 스크립트

## 1. 문제 정의
“이 프로젝트는 개인화 추천 시스템을 실제 서비스처럼 구현한 포트폴리오입니다. 인기순 추천이 아니라 사용자별로 다른 추천 결과를 만들고, 정확도뿐 아니라 다양성과 카탈로그 커버리지까지 함께 평가하는 것을 목표로 했습니다.”

## 2. 아키텍처
“구조는 세 단계입니다. 먼저 협업 필터링 기반 latent retriever가 수백 개 후보를 빠르게 뽑고, 그다음 ranker가 사용자/아이템 메타데이터와 후보 점수를 이용해 재정렬합니다. 마지막으로 MMR 기반 diversity reranker가 같은 장르가 과도하게 몰리지 않게 조정합니다.”

## 3. 실행 화면
- `/` 대시보드 접속
- 기존 사용자 ID 입력 후 추천 요청
- 추천 카드의 장르와 추천 이유 확인
- 콜드스타트 추천 폼으로 선호 장르만 넣고 추천 시연

## 4. 평가 포인트
“보고서는 Popularity, Latent Retriever, Final Multi-Stage 세 모델을 비교합니다. 여기서 Final 모델이 relevance와 catalog metric의 균형을 보여주는 것이 핵심입니다.”

## 5. 마무리
“추천 시스템은 단순 모델 성능보다 데이터 분할, 후보 생성 전략, 재랭킹, 콜드스타트, 운영 지표가 더 중요하다고 생각해서 전체 파이프라인 관점으로 구현했습니다.”
